# Social Welfare Exam Study System - 仕様書

本システムは、社会福祉士・精神保健福祉士国家試験の学習を効率化するために開発された、AI駆動型のデスクトップアプリケーションです。

## 1. テクノロジースタック
- **Core**: [Tauri](https://tauri.app/) (Rust + React + Vite)
- **Database**: SQLite (Local storage)
- **AI Engine**: Google Gemini 2.0 Flash API
- **UI Libraries**: Tailwind CSS, Shadcn UI, Recharts (グラフ描画), Lucide-React (アイコン)

## 2. 主要機能
### 2.1 ユーザー管理
- **マルチユーザー対応**: 名前ベースで複数のプロフィールを作成可能。
- **セッション復元**: 各学習モード・カテゴリごとに、最後に解いていた問題を自動記憶し、次回再開時に復元。

### 2.2 学習システム
- **習得度(Mastery)システム**: 
  - 5回連続正解で「Mastered（習得済み）」と判定。
  - 忘却曲線に基づいた復習スケジュール（2の累乗日後に再出題）を自動計算。
- **回答履歴**: 全回答のタイムスタンプと正誤を記録。
- **複数回答への対応**: 「2つ選べ」などの複数選択問題に対応。

### 2.3 データ可視化（ダッシュボード）
- **トータル進捗**: 全体およびカテゴリ別の習得率を表示。
- **学習アクティビティ**: 直近14日間の解答数をエリアチャートでグラフ表示。
- **カテゴリ別統計**: 科目ごとの進捗状況を一目で確認可能。

### 2.4 問題生成・インポート
- **PDF学習**: 試験問題や資料のPDFを読み込み、AIが「問題・選択肢・解答・解説」を自動生成。
- **一括処理（Bulk Import）**: 複数のPDFファイルを一括で問題化。
- **重複排除機能**: すでにデータベースにある問題と似た内容の重複登録を防止。

### 2.5 共有・同期機能
- **問題パック（Question Pack）**:
  - 生成した問題をJSON形式でエクスポートし、他ユーザーと共有可能。
- **ユーザーデータ同期**:
  - 進捗データのバックアップ（JSON）およびインポートに対応。
  - 指定したエンドポイントへのクラウド同期プロトコルを装備。

## 3. データベース構造 (SQLite)
- `questions`: 問題本体、解説、カテゴリ、正解（JSON配列）
- `users`: ユーザー名
- `user_progress`: ユーザーごとの習得度、連続正答数、次回復習日
- `learning_history`: 全回答のログ
- `user_sessions`: モードごとの最終閲覧問題ID

## 4. 製品化（有料化）に向けた考慮事項
- **オフライン動作**: 一度生成した問題はローカル環境で恒久的に利用可能（APIコスト不要）。
- **資産化**: エクスポートされたJSONデータは、そのまま製品版の初期コンテンツとして利用可能。
- **スケーラビリティ**: マルチユーザー、セッション管理、複数回答対応など、商用アプリとしての基本機能を完備。

## 5. 運用マニュアル・新機能詳細

### 5.1 AI監査・洗練機能 (AI Audit & Polish)
データベースに蓄積された問題を AI (Gemini 2.0 Flash) が総点検し、品質を向上させる機能です。

*   **機能**:
    *   問題文の事実関係チェックと修正（法改正対応など）。
    *   解説文の充実化・ブラッシュアップ。
    *   選択肢の論理的整合性の確認。
    *   単一選択形式で登録されている複数回答問題の検出と修正。
*   **実行方法**:
    1.  アプリの「Importer」メニューを開く。
    2.  最下部の **「✨ RUN AI AUDIT & POLISH DATABASE」** ボタンをクリック。
*   **変更履歴 (Audit Logs)**:
    *   修正が行われた場合、その差分は以下のディレクトリにCSVファイルとして自動保存されます。
    *   保存先: `(アプリのルート)/audit_logs/audit_log_YYYYMMDD_HHMMSS.csv`
    *   CSVには「問題ID」「変更箇所」「修正前の内容」「修正後の内容」が記録されます。

### 5.2 複数回答問題 (Multiple Choice)
「正しいものを2つ選べ」といった複数選択形式の問題に完全対応しています。

*   **仕様**:
    *   データベース上の `correct_answer` カラムには、正解の選択肢配列が JSON 形式で格納されます（例: `["選択肢A", "選択肢C"]`）。
    *   **重要**: 古い形式（単一文字列の正解データ）も互換性維持のため引き続き読み込み可能です。
*   **操作**:
    *   正解が複数ある問題では、学習画面で複数の選択肢をクリックして選択状態にし、「SUBMIT ANSWERS」ボタンを押して解答します。

## 6. アプリケーションのビルドと配布
本アプリを配布用（インストーラー）としてパッケージ化する手順です。

### 6.1 前提条件
*   Node.js および Rust (Cargo) がインストールされていること。
*   `gemini-2.0-flash` が利用可能な API キーが環境変数等で設定されていること（開発・ビルド時）。

### 6.2 ビルドコマンド
ターミナルで以下を実行します。
```bash
npm run tauri build
```

### 6.3 生成物の場所
ビルド成功後、以下のディレクトリにインストーラーが生成されます。
*   **Windows (インストーラー)**:
    `src-tauri/target/release/bundle/nsis/social-welfare-app_0.1.0_x64-setup.exe`
*   **Windows (MSI)**:
    `src-tauri/target/release/bundle/msi/social-welfare-app_0.1.0_x64_en-US.msi`

配布時は、この `.exe` ファイルをユーザーに提供してください。製品化の際は、事前に「AI Audit」を実行してデータベースを完成させた状態でビルドに含める（または初期データJSONとして同梱する）ことを推奨します。
